# Beyond the Original — 5つの拡張画面提案

公式アプリの再現を超えて、来場体験の **「来る前 → 入る → 巡る → 思い出す」** までを通しでカバーするために5画面を追加提案する。
すべて既存のデザインシステムだけで組めるので、追加トークンは要らない。

---

## 1. Onboarding（来場前 / 初回起動時）

**目的**：会場に入る前に、音声・振動・言語の設定を済ませてもらう。会場内で初めて設定画面を出さない。

| 要素 | 内容 |
|---|---|
| 画面構成 | 3ステップ：Audio / Haptics / Language |
| デザイン | 黒背景＋中央に Water Blue 球体（光のメタファー）＋短文 |
| コピー | "Quiet guide for immersive space" / "Three short steps to set up" |
| ナビ | 進行バー（左下）／ Continue ボタン（右下） |
| Skip | いつでも skip 可能（右上） |

**評価される観点**：会場内で迷わない事前準備、片手操作、Quiet Mode への自然な導線。

---

## 2. Route Plan（来場直後）

**目的**：8作品 1時間20分 のような「今日の動線」を最初に提示し、来場者が回遊計画を立てやすくする。

| 要素 | 内容 |
|---|---|
| 画面構成 | 縦のタイムライン（1〜8）／所要時間・距離 |
| ロジック | 現在の混雑データに基づいた "Less crowded path" を初期提案 |
| カスタマイズ | 各作品の追加・削除 / 並べ替え |
| CTA | "Start Route" → Quiet Mode と紐づく |

**評価される観点**：UX提案の実装可能性、混雑分散という運営側の利点、来場者の認知負荷低減。

---

## 3. Quiet Mode（鑑賞中・課題2の中心）

**目的**：作品体験を邪魔しない、低輝度・少情報・振動中心モード。
（詳細は §補足資料.md / §提出パッケージ_完全版.md に記載済み）

差別化のポイント：
- **ナレーションを自動再生しない**（鑑賞の集中を切らさない）
- **方向・距離・作品名のみを画面に残す**
- **触覚で位置情報を伝える**

---

## 4. Memory Album（帰り道 / 帰宅後）

**目的**：来場の余韻を、押し付けがましくない形で持ち帰る。

| 要素 | 内容 |
|---|---|
| 画面構成 | 訪問数 / 時間 / 訪れた作品サムネのグリッド |
| 共有 | "Generate a quiet photo album" — 作品名を付けた静かな共有用画像を作る |
| プライバシー | 位置情報や顔は含まない |

**評価される観点**：体験を1回で終わらせないリテンション、SNS共有時にも brand トーンが崩れない設計。

---

## 5. Accessibility Mode

**目的**：「すべての来場者が等しく楽しめる」というチームラボの姿勢に応える。

| 要素 | 内容 |
|---|---|
| High contrast text | ON/OFF |
| Text size | 100–150% スライダー |
| Audio description | 音声ガイドを副音声に切替 |
| Reduce motion | システム設定と連動 |
| Staff assist banner | 「Show this screen to staff」— スタッフが介助の必要性を視認できる画面 |

**評価される観点**：商用アプリとしての配慮、ローンチ後の運用負荷を減らす設計。

---

## なぜこの5画面なのか（面接で言う一言）

> 「公式アプリは展示会場の中で迷わないことに集中していました。一方で、来場体験は会場の中だけで完結しないと考え、**来場前のセットアップ・回遊計画・鑑賞中のQuiet Mode・帰宅後の余韻・誰でも使えるアクセシビリティ** の5層を、既存のデザインシステムを拡張せずに足しました。新しいトークンを追加していないので、デザインシステムの一貫性を崩していないつもりです。」
