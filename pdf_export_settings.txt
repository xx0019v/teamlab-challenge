============================================================
elevated_set_12screens.html → PDF 化の手順
============================================================

このPDFは、課題2「展示会場におけるユーザー体験の提案」の
補足資料として、再提出メールに添付するものです。

完成すると7ページのエディトリアル資料になります：
01 Cover / 02 Manifesto / 03 Experience Flow /
04 Design System / 05 12 Screens / 06 What Changed /
07 Closing


■ 推奨ブラウザ

Google Chrome
（Safari でも可。ただし Google Fonts と背景色は Chrome が一番安定）


■ 手順

1. elevated_set_12screens.html を Chrome で開く
   - ローカルファイル直接：
     /Users/exx/Tsc/teamlab-challenge/elevated_set_12screens.html
   - またはローカルサーバー経由：
     http://localhost:8765/elevated_set_12screens.html

2. Cmd + P を押す（印刷ダイアログが開く）

3. 設定（重要）：

   送信先          ：PDF に保存
   ページ          ：すべて
   用紙サイズ      ：A3
   向き            ：横（横長 / Landscape）
   ページ数        ：1
   余白            ：既定（または「なし」）
   倍率            ：既定（100%）
                     1枚に収まらない場合のみ 90〜95%
   ヘッダーとフッター：オフ
   背景のグラフィック：オン  ← 最重要。これを忘れると黒背景が消える

4. 「保存」ボタンを押す

5. ファイル名：
   teamLab_Elevated_Set_アヴェンダニョシンタロウ.pdf

6. 保存先：デスクトップなど分かりやすい場所


■ 完成後の確認ポイント

[ ] 7ページになっている
[ ] Cover の黒背景が真っ黒に出ている
[ ] Water Blue の淡いグローが消えていない
[ ] セリフ書体（Cormorant Garamond）が表示されている
[ ] 12画面のスマホフレームが小さすぎない
[ ] 日本語のテキストが文字化けしていない
[ ] 「Quiet Guide」「Artwork Detail」「Exhibition Map」が大きめに見える
[ ] Closing の余韻メッセージが綺麗に出ている


■ A3 が選べない環境の場合

A4 横向きでも提出可能。
12画面のグリッドが4列に折り返すフォールバックを CSS に
入れてあるので、レイアウトは崩れません。
ただし、文字が小さくなるので、可能なら A3 を推奨します。


■ よくあるトラブル

「黒背景が白くなる」
 → 「背景のグラフィック」のチェックを入れ忘れている。
   Chrome の印刷ダイアログ左下「詳細設定」を開いて確認。

「英語の見出しが普通のサンセリフになる」
 → ネット環境を確認。Google Fonts が読み込めていない可能性。
   Wi-Fiを確認して再読み込み（Cmd + R）してからPDF化し直す。

「ページが意図せず増える / 減る」
 → 倍率を100% → 90% に下げてみる。
   それでも改善しない場合は、ブラウザを一度再起動。

「日本語が文字化け／豆腐になる」
 → Noto Sans JP が読み込まれていない可能性。
   ネット接続を確認、または Chrome のキャッシュをクリア。
