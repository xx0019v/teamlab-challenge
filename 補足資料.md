# teamLab Guide App
## UI/UX Redesign — Design System and Quiet Guide Mode Proposal

アヴェンダニョ シンタロウ
2026.06.15

---

## 1. はじめに

本資料は、チームラボ展示会場で使用される公式ガイドアプリを対象に、提供デザインの再現と、開発・運用しやすいデザインシステムの構築、そして会場環境に適したUX改善提案をまとめたものです。

「見た目を真似る」ことよりも、**他のデザイナーや開発者が触ったときに迷わないデータ構造**、そして**展示会場という固有の利用環境にきちんと答えるUI**を目指しました。

---

## 2. 課題の理解 — Understanding the Challenge

本課題で求められているのは、提供画像の再現に留まらず、

- Auto Layout を前提とした柔軟な構造
- コンポーネント・スタイル・余白ルールの整理
- 展示会場（暗所・回遊・没入）という固有の文脈への配慮

の3点だと捉えました。
このため、最初に **デザインシステムを定義し、その上で画面を組み立てる** という順序で進めました。1画面ごとに調整するのではなく、トークン → コンポーネント → 画面、という階層を保っています。

---

## 3. 利用環境 — Use Environment

チームラボの展示会場には次のような特徴があります。

- **暗所**：明るい画面は周囲の作品体験を妨げる。眩しさで瞳孔が縮み、作品が見えにくくなる。
- **回遊**：来場者は歩きながらスマホを確認する。立ち止まって長く読む状況には適さない。
- **身体的没入**：作品に対する集中を切らさないことが体験価値そのもの。

このため、本デザインは **「画面を見る時間を短くする」「光量を抑える」「片手で完結する」** という3つの制約を常に意識しました。

---

## 4. デザインコンセプト — Quiet Guide for Immersive Space

> 没入空間を邪魔しない、静かで直感的な展示ガイドUI。

このアプリは作品の主役ではありません。
作品に集中するユーザーが、必要な時にだけ、必要な情報だけを受け取れる──そういう「背景」のような存在を目指しました。

ビジュアル方針：

- 背景は深い黒・ダークグレー
- 文字は白を基調に、重要度を透明度で表現
- アクセントは水と光を感じる Water Blue (#69D9FF)
- 装飾を最小限にし、余白・階層・コントラストで構造を伝える

---

## 5. デザインシステム — Design System

### 5.1 構造の考え方

| Layer | 内容 |
|---|---|
| Token | Color / Typography / Spacing / Radius / Shadow |
| Component | Button / Card / Nav / Sheet / Tag など 11種 ＋ Variant |
| Pattern | Topbar+Hero / List+Tag / Sheet展開 など |
| Screen | 9画面（Splash〜Settings） |

### 5.2 主要トークン（抜粋）

- 背景は3階層（Black / Dark Gray / Card）で奥行きを表現。
- 文字は不透明度で4階層（100/72/48/32%）。
- アクセントは Water Blue 一色のみを許容し、装飾色は持たない。
- Spacingは8ptグリッド、画面マージンは24pt固定。
- 角丸は4段階（8/16/24/full）に統一。

### 5.3 開発を意識した点

- すべての色を **Color Variable** で管理し、Light/Dark への将来拡張に備えた構造にしています。
- すべての文字は **Text Style** 経由で適用し、画面ごとに直接フォント指定をしません。
- すべての画面に **Auto Layout** を適用しているため、内容差し替え時に手作業の再調整が不要です。
- 命名は英語snake_caseで統一し、エンジニアがCSSやSwiftUIで再現する際の翻訳コストを減らしました。

## 6. UI再現の考え方 — UI Reproduction

公式アプリのHomeはエリア画像カードを縦に積み、上部にハンバーガーメニューと中央タイトル、下部に浮動の「現在地」カードを置く構造でした。これは来場者がまず「どこにいるか」を直感的に掴むための設計だと捉え、構造そのものは保持しています。

その上で、暗所での視認性と次の行動への導線を強化するために次の調整を加えました。

- **作品ビジュアルが画面の主役**：エリアカードは画像比率を最大化し、テキスト面積を抑える。
- **eyebrow ラベルの追加**：各カード上部に「YOU ARE HERE」「FLORAL · 5 ROOMS」など、短い導入ラベルを置いて階層を明確化。
- **glass blur 化したTopbar と Floating Card**：装飾を増やさずに階層を伝える。
- **音声ガイドの Component 化**：Detail 画面の音声プレーヤーを独立した Component に格上げし、操作領域と階層を整える。
- **暗所の可読性**：本文は最小14pt、見出しは20pt以上、白＋透明度4階層、コントラスト4.5:1以上。
- **誤タップ防止**：すべてのインタラクティブ要素を44pt以上、間隔は8pt以上確保。
- **片手操作**：主要CTA は画面下半分、戻る／メニューは左上に集約。

## 7. UX提案 — Quiet Guide Mode

### 7.1 課題

通常のアプリUIは、明るい画面と多めの情報量を前提に作られています。これを暗所の回遊体験にそのまま持ち込むと、

- 画面の光が周囲の作品体験を侵食する
- 文章を読むために立ち止まる必要が生じる
- 細かい操作で来場者の意識がスマホに引き戻される

という、作品体験と相性の悪い状況が起こりがちです。

### 7.2 提案

会場専用のモードとして **「Quiet Guide Mode」** を提案します。
これは、低輝度・少情報・振動中心のガイドモードで、作品の没入を維持したまま会場を移動できるようにする機能です。

### 7.3 主な要素

1. **低輝度UI**：背景は完全な黒。文字色は通常モードよりさらに透明度を上げる。
2. **方向案内優先**：地図ではなく「次に向かう方向」を矢印と距離だけで提示。
3. **短文ガイド**：作品の説明文ではなく、一言の体験ヒントのみ。
4. **振動フィードバック**：曲がる・近づく・混雑を避ける、を振動パターンで伝える。
5. **混雑回避ルーティング**：空いている作品を優先的に提案する。

### 7.4 ユーザー体験フロー

1. 来場者が Home から Quiet Guide を起動
2. 現在地から近い作品が1つだけ表示される
3. 画面には次の方向、距離、作品名だけが残る
4. 接近すると軽い振動と短い体験ヒントが現れる
5. 鑑賞後、次の作品候補に自然に切り替わる

### 7.5 期待される効果

- 画面の光が作品体験を妨げにくくなる
- 歩きながらでも一目で次の行動が分かる
- 文章を読む時間が減り、来場者の意識が作品側に戻る
- 混雑が分散され、回遊体験全体の質が上がる

本提案では、提供デザインを再現するだけでなく、

- 開発・運用を見据えた **構造化されたデザインシステム**

の2点を中心にまとめました。
アプリ自身が主役になるのではなく、作品体験を背景から支える存在になることを目指しています。
