01

The brief.

Reproduce the provided design — but aim for data that other people can build on, not a screenshot.

teamLab's exhibition guide app was provided as a finished Figma design. The task: rebuild it in a design tool so the structure, not just the surface, survives the handoff.

チームラボの展示ガイドアプリが完成デザインとして提供された。課題は、表面だけでなく「構造」が受け渡しに耐える形で再現すること。

02

Three rules for an honest reproduction.

01

Fidelity, not invention

Keep the original structure — dark theme, full-bleed artwork, the floating "you are here" card, the audio player. The reproduction earns trust before anything is improved.

元の構造を保つ。再現は、改善する前にまず信頼を得る。

02

Tokens, not values

Every colour, type style, spacing and radius lives as a named style — never a raw value pasted into a screen. The system is the deliverable.

色・文字・余白・角丸は名前付きスタイルで管理。画面に直接値を書かない。

03

Named for engineers

Auto Layout everywhere, variants for states, English layer names that map straight to code. The data should read like documentation.

Auto Layout と Variant、コードに直結する英語命名。データ自体がドキュメントになる。

05

Built like a component library.

Screen / Artwork Detail ├─ StatusBar / component ├─ Topbar back · title · menu ├─ artwork / group │ ├─ base.gradient │ ├─ streaks │ ├─ particles.far blur │ └─ particles.near ├─ scrim ├─ content / auto-layout │ ├─ title │ ├─ meta │ └─ body ├─ AudioPlayer / component └─ HomeIndicator

Reusable by design.

  • Auto Layout, everywhere. Cards, lists and the audio player hug their content, so swapping an artwork never breaks the layout.
    中身を差し替えても崩れない。
  • Variants for states. Visited / nearby / current are properties on one component, not separate copies.
    状態は1コンポーネントのVariantで管理。
  • English, code-ready names. Screen / [Name] / [State] maps straight to a developer's file.
    命名がそのまま実装の手がかりになる。
  • One source of truth. Change a token, every screen updates — the system, not the screen, holds the value.
    トークンを変えれば全画面に反映。
