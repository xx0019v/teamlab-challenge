01 / CONCEPT

アプリは、作品体験の
背景に徹するべきだ。

チームラボの展示会場には、暗所・回遊・身体的没入という3つの強い特徴があります。 通常の明るく情報量の多いアプリUIは、瞳孔を狭め、立ち止まる時間を増やし、没入を切らす要因になりがちです。 本提案では、アプリが主役になるのをやめて、必要な情報だけを最小限の光と短い時間で伝える設計に切り替えます。

PRINCIPLE 01

暗所での視認性を最優先する

背景は深い黒、文字は白を透明度で4段階に分解。アクセントは Water Blue 1色のみに絞り、装飾色を持たない。本文は14pt未満を使わない。

PRINCIPLE 02

歩きながら片手で完結する

すべてのインタラクティブ要素を44pt以上、間隔は8pt以上確保。主要CTAは画面下半分。戻る／メニューは左上に集約し、視線移動を最小化する。

PRINCIPLE 03

画面を見る時間そのものを減らす

Quiet Mode では情報を方向・距離・作品名だけに絞り、案内は振動で補う。ナレーションは自動再生しない。アプリは「読むもの」ではなく「合図」に。

02 / DESIGN SYSTEM

Tokens, not values.

色・タイポグラフィ・余白・角丸をすべて Variable と Style 経由で管理。画面側で hardcoded された値を持たない構造にしました。
