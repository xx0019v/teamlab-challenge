# 5. 課題2 UX提案文章（Quiet Guide Mode）

### 課題

チームラボの展示会場には「暗所・回遊・身体的没入」という3つの特徴があります。明るく情報量の多い通常のアプリUIは、来場者の瞳孔を狭め、作品の光や色が見えにくくなる原因になります。また、歩きながら長い文章を読むことや、複雑な操作はそのまま没入感を切る要因になります。

### 提案

会場専用の **「Quiet Guide Mode」** を提案します。
これは、**低輝度・少情報・振動中心** で次の作品へ案内するモードで、来場者がスマホを見る時間そのものを最小化することを目的としています。

### 主な機能

1. **低輝度UI**：背景は完全な黒、文字は通常モードよりさらに透明度を上げて表示。
2. **方向だけを示す案内**：地図ではなく、次に向かう矢印と距離だけを大きく表示。
3. **短文ガイド**：長い解説は出さず、作品ごとに一言の体験ヒントだけ。
4. **振動フィードバック**：曲がる／近づく／混雑回避 を振動パターンで伝える。
5. **混雑回避ルーティング**：空いている作品を優先的に提案する。

### ユーザー体験フロー

1. Home から Quiet Guide Mode を起動
2. 現在地に近い作品 1件 が表示される
3. 画面には方向・距離・作品名のみ
4. 近づくと軽い振動と短い体験ヒント
5. 鑑賞後、次の作品候補に自然に切り替わる

### 期待される効果

- 画面の光が作品体験を妨げにくくなる
- 歩きながらでも一目で次の行動が分かる
- 文章を読む時間が減り、意識が作品側に戻る
- 混雑が分散され、回遊全体の質が上がる
- アプリと展示体験が断絶せず、自然につながる

### Page 1 — Title

**teamLab Guide App UI/UX Redesign**
*Design System and Quiet Guide Mode Proposal*

本資料では、チームラボ展示会場で使用される公式アプリを想定し、UIの再現だけでなく、開発・運用しやすいデザインシステムの構築と、展示会場の環境に適したUX改善提案をまとめました。

---

### Page 2 — Understanding the Challenge

本課題で重要なのは、見た目を再現することそのものではなく、**他のデザイナーや開発者が触ったときに迷わないUIデータ**を作ることだと捉えました。

そのため、Auto Layout を前提に Color / Text Style / Component を最初に定義し、その上に画面を組み立てる順序で進めています。各画面ごとに値を直接指定するのではなく、トークン → コンポーネント → 画面 という階層を維持することで、再利用性と拡張性を持たせています。

---

### Page 3 — Use Environment

チームラボの展示会場には「暗所」「回遊」「身体的没入」という3つの特徴があります。来場者は作品を見ながら移動し、必要な時にだけアプリを確認します。

このため、本デザインでは次の3つの制約を最初に置きました。
1. 画面を見る時間そのものを短くする
2. 光量を最小限に抑える
3. 片手・歩行中に操作が完結する

アプリは情報を主張する存在ではなく、作品体験を背景から支える存在であるべきだと考えました。

---

### Page 4 — Design Concept

**Quiet Guide for Immersive Space**
没入空間を邪魔しない、静かで直感的な展示ガイドUI。

光・水・余白を感じるトーンを保ちながら、暗い会場でも読みやすく、片手で操作しやすいUIを目指しました。背景は深い黒、文字は白を基調に透明度で階層を表現し、アクセントは水と光を感じる Water Blue（#69D9FF）一色のみに絞っています。装飾を増やすのではなく、余白・階層・コントラストで構造を伝えることを優先しました。

---

### Page 5 — Design System

カラー・タイポグラフィ・余白・角丸・コンポーネントを整理し、画面ごとの再利用性を高めました。色は Variable で管理し将来の Light/Dark 拡張に備え、文字はすべて Text Style 経由で適用しています。

特に Artwork Card、Bottom Navigation、Button、BottomSheet は複数画面で使い回せる構造になっており、命名は英語snake_caseに統一して、エンジニアが実装に翻訳する際のコストを下げることを意識しました。

---

### Page 6 — UI Reproduction

提供されたデザインのトーン（作品画像が主役、控えめな文字、深いコントラスト）を尊重しながら、スマホ画面として自然に機能する構成に再整理しました。

- 作品ビジュアルが画面の主役、テキストは最小限
- 本文は14pt以上、見出しは20pt以上、コントラストは4.5:1以上を維持
- インタラクティブ要素はすべて44pt以上、間隔は8pt以上
- 主要CTAは画面下半分、戻る／メニューは左上に集約

---

### Page 7 — UX Proposal: Quiet Guide Mode

展示会場では、スマートフォンの明るさや情報量が作品体験を侵食する可能性があります。

そこで会場専用の低輝度・少情報・振動中心モードとして **Quiet Guide Mode** を提案します。画面には方向・距離・作品名のみを残し、解説は短い体験ヒントに絞り、案内は振動で補います。

このモードにより、来場者は作品への没入感を保ったまま、迷わず次の展示へ移動できます。混雑時には空いている作品を優先して提案するため、回遊全体の質も改善します。

---

### Page 8 — Conclusion

本提案では、視覚的な再現だけでなく、

- 構造化されたデザインシステム
- 会場環境に応えた UX改善（Quiet Guide Mode）

の2点を重視しました。

アプリが主役になるのではなく、作品体験を自然に支える存在になることを目指しています。ご確認のほど、よろしくお願いいたします。

*アヴェンダニョ シンタロウ*
