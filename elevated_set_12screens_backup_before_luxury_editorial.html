01 / CONCEPT

An app that does not compete with the artwork,
but quietly supports the visitor.

アプリは体験の背景に徹し、必要なときだけ来場者を導くべきだと考えました。
暗く、没入的で、身体的な空間において、UIは視覚的なノイズを減らし、自然な移動を支える存在であるべきです。

PRINCIPLE 01

Low Brightness

暗い展示空間に合わせて、UI全体を低輝度・低彩度で構成。背景は深い黒、アクセントは Water Blue 1色のみに絞り、装飾色は持たない設計にしました。

PRINCIPLE 02

Less Text During Immersion

来場者が作品を見ている間、アプリは情報量を絞ります。画面上には方向・距離・短いヒントだけを残し、意識が作品側に戻るように設計しました。

PRINCIPLE 03

Movement-first Guidance

大きなタップ領域、短い方向指示、振動による合図で、歩きながらの来場者を支えます。片手・移動中・低照度の環境で使えることを前提に設計しました。

02 / EXPERIENCE FLOW

Five quiet moments,
from anticipation to memory.

展示体験は入口で始まり、出口で終わるわけではありません。自宅で準備をはじめ、訪問後にも余韻として続きます。本提案ではアプリを5つの瞬間にマッピングし、各画面が「なぜ存在するか」を明確にしました。

Stage 01

Before Visit

チケットと基本情報を、当日の前に静かに整える。

Stage 02

Entering

入口での迷いを減らし、来場者の意識を作品側に渡す。

Stage 03

Exploring

近くの作品を、押しつけがましくなく自然に案内する。

Stage 04

Immersing

作品を体験している間、画面は静かなままでいる。

Stage 05

Remembering

来場後に静かに振り返り、余韻を持ち帰れるようにする。

03 / DESIGN SYSTEM

Tokens, not values.

色・タイポグラフィ・余白・角丸は、画面内の生の数値ではなく Variable と Style で管理しています。Auto Layout を前提に12画面で再利用し、命名は英語で統一して、開発側がコードに翻訳しなくても扱える構造にしました。Figma本体（課題1）のデザインシステムの延長として位置づけています。

05 / WHAT CHANGED

Lower Brightness
深い黒の背景と最小限のアクセントで、暗い空間でも画面が作品と競合しないように設計しました。
Less Text During Immersion
作品の前に立っている間、Quiet Guide は方向・距離・短いヒントだけに情報を絞ります。
Larger Tap Areas
すべてのインタラクティブ要素を44pt以上、間隔8pt以上に確保。歩きながらでも誤タップしにくい設計です。
Vibration-based Guidance
接近・曲がる・到着を、音や長文ではなく短い振動パターンで伝え、視線を画面に張り付けません。
Crowd-aware Routing
Crowd Balance がリアルタイムの混雑度を読み取り、空いているルートを提案して来場者を会場全体に分散します。
Reusable UI Components
Topbar / Audio Player / Floating Current / Bottom Sheet / Button / Tag / Map Pin を12画面で共通利用しています。
Structured Design System
色・タイポ・余白・角丸・モーションを Variable と Style で管理し、画面ごとに値を直書きしない構造にしました。
Visitor-first Hierarchy
各画面に1つの明確な意図を持たせ、最重要のアクションは常に親指の届く範囲に配置しています。
